# OSCAR stack

End-to-end install for OSCAR on top of a ServiceBay full-stack host.

A ServiceBay stack is **documentation-only** — there's no programmatic "deploy these four" button. You walk the wizard four times, in the order below. Each individual `oscar-*` template appears in the ServiceBay wizard once the OSCAR registry is added.

## Prerequisites

- **ServiceBay v3.16+** on a Fedora CoreOS host with the **full-stack** deployed (auth, nginx, home-assistant, …).
- **[mdopp/servicebay#348](https://github.com/mdopp/servicebay/issues/348)** merged so the HA pod can be deployed with `VOICE_BUILTIN=disabled` (otherwise Wyoming ports collide with `oscar-voice`).
- **[mdopp/servicebay#443](https://github.com/mdopp/servicebay/issues/443)** merged so ServiceBay can sync external git registries (its container needs `git`).
- **HA-MCP** integration enabled in your Home Assistant (Settings → Devices & Services → Add Integration → "Model Context Protocol Server").
- A **ServiceBay-MCP** bearer token (Settings → Integrations → MCP → Generate token, scope `read+lifecycle`).
- For **gpu-local** mode: `nvidia-container-toolkit` + CDI on the host. For **cpu-local** / **cloud** modes: nothing extra.

## Step 0 — Add the OSCAR registry

ServiceBay → Settings → Registries → Add:

- Name: `oscar`
- URL: `https://github.com/mdopp/oscar.git`

After save, the four `oscar-*` templates appear in the wizard.

## Step 1 — Deploy `oscar-brain` (data plane)

Postgres + Qdrant + Ollama + alembic-driven migrations + weekly pg_dump backup sidecar.

ServiceBay wizard → `oscar-brain` → fill in:

- `POSTGRES_PASSWORD` (auto-generated)
- `OLLAMA_ENABLED=yes` for local LLM (default), empty for cloud-only
- `GPU_PASSTHROUGH=yes` if you have an NVIDIA GPU with CDI

Deploy. First start takes 5–10 min while Ollama pulls the default model.

## Step 2 — Deploy `oscar-hermes` (agent runtime)

Wraps the official `docker.io/nousresearch/hermes-agent` container as a ServiceBay pod. This is where conversation, gateways, cron, skills, and MCP-client integration live.

ServiceBay wizard → `oscar-hermes` → fill in:

- `API_SERVER_ENABLED=yes`
- `API_SERVER_KEY` — auto-generated by the wizard if blank
- `HERMES_DASHBOARD=` (empty) — only enable later over SSH tunnel
- `OSCAR_REGISTRY_DIR` — leave default unless your ServiceBay clone is elsewhere

Deploy. The pod runs `gateway run`, which idles until you do the one-time setup:

```bash
ssh <oscar-host>
podman exec -it oscar-hermes-hermes hermes setup
# pick LLM provider (Ollama at http://localhost:11434 or a cloud key) and write the .env
# then connect MCP servers:
podman exec -it oscar-hermes-hermes hermes mcp add http://localhost:8123/mcp_server/sse --token <HA-LLAT>
podman exec -it oscar-hermes-hermes hermes mcp add http://localhost:5888/mcp           --token <SB-MCP-token>
# pair a messaging gateway:
podman exec -it oscar-hermes-hermes hermes gateway setup signal
```

Restart the pod once so `gateway run` picks up the new `.env`:

```bash
systemctl --user restart oscar-hermes.service
```

OSCAR's repo-side `skills/` is read-mounted at `/opt/data/skills/oscar`, so the household-specific skills (light, status, audit-query, debug-set) show up alongside Hermes' built-in Skills Hub.

## Step 3 — Deploy `oscar-voice` (optional, when you have HA Voice PE devices)

Wyoming pipeline + gatekeeper. The gatekeeper calls back into oscar-hermes for the conversation step (`HERMES_URL=http://127.0.0.1:8642` by default — both pods use hostNetwork).

ServiceBay wizard → `oscar-voice` → set variables to match your hardware (`STT_GPU_PASSTHROUGH=yes` for gpu-local).

If a `voice` service is already running on the node (ServiceBay's built-in voice template), **delete it first** (`Settings → Services → voice → Delete`) — it occupies the same Wyoming ports (10300/10200/10400).

## Step 4 — Deploy `oscar-connectors` (optional, when you want connector-MCP)

Weather + cloud-LLM-with-audit. Hermes picks them up via `hermes mcp add`:

```bash
podman exec -it oscar-hermes-hermes hermes mcp add http://localhost:8801 --token <CONNECTORS_BEARER>
podman exec -it oscar-hermes-hermes hermes mcp add http://localhost:8802 --token <CONNECTORS_BEARER>
```

## After install

Talk to OSCAR through whichever gateway you paired with Hermes:

```
You (Signal):    bist du da?
OSCAR:           ja, alles ok. ollama, postgres, ha-mcp und servicebay-mcp antworten.
You:             mach das wohnzimmerlicht an
OSCAR:           ok, wohnzimmer ist an.
You:             stell einen timer auf 5 minuten
OSCAR:           ok, fünf minuten.
```

Light control routes through HA-MCP (the `oscar-light` skill). Timer/alarm/reminder use Hermes' native cron scheduler. Status checks use `oscar-status`.
